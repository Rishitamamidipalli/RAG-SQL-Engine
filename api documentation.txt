API Documentation
API Endpoint

https://aicafe.hcl.com/AICafeService/api/v1/subscription/openai/deployments/{deploymentName}/chat/completions?api-version={apiVerion}
Method
POST
Description

You interact with ChatGPT models via the https://aicafe.hcl.com/AICafeService/api/v1/subscription/openai/deployments/{deploymentName}/chat/completions?api-version={apiVerion} endpoint. The request format is typically JSON with the following structure (to be sent as the request body):

Request Parameters
Name	Type	Required	Description	Location
deploymentName	String	Yes	The name of the model deployment to use for this request. Set model value as "gpt-4.1".	Path Variable
api-version	String	Yes	Specifies the version of the API to use for the request. Set api-version as "2024-12-01-preview".	Query
api-key	String	Yes	Subscription key for the API, which is required for authentication.	Header
model	String	Yes	Specify which deployed model you want to use for generating completions, chat responses, embeddings, etc.	Body
maxTokens	Number	No	Specify the maximum number of tokens that the model should generate in its response (completion).	Body
temperature	Number	No	The temperature parameter controls the randomness or creativity of the model's responses.	Body
messages	Array	Yes	An array of message objects representing the conversation history. Each object should have a 'role' (such as 'user', 'assistant', or 'system') and 'content' (the text of the message).	Body
Subscription Key Example

{
  "api-key": "your-api-subscription-key-here"
}
Request Example

{
  "model": "gpt-4.1",
  "messages": [
    {
      "role": "user",
      "content": "What is React?"
    }
  ],
  "maxTokens": 100,
  "temperature": 0.7
}
